## 类别 A:信贷管理系统

### A-1:同盾科技 智能信贷(零售信贷风险管理)
- **定位**:面向银行/消金/互联网金融的贷前贷中贷后全流程风控中台,服务 10000+ 企业
- **核心功能**:进件管理、决策引擎(规则+模型+决策流)、模型中心(部署/版本/监控)、贷后审查、核查中心人工审核、对接中心
- **架构**:分布式 + 微服务,单节点 3000+ TPS,Kie/Drools 衍生决策流,国产化(鲲鹏/海光/TiDB)适配
- **可借鉴点**:①"进件→审查→贷后"三段式状态机对应"项目→议案→材料" ② 决策流可视化配置类比"规则引擎触发" ③ 审查中心(人审)对应我们"投委会审议"
- **不适用点**:高并发交易反欺诈(我们 50 人规模用不上);三方征信/反欺诈模型(本地部署买不到)
- **评分**:业务匹配 3 / 技术栈 4 / 可借鉴 4 / 实现成本 3 = **14/20**
- **来源**:https://同盾科技/product/loan/before | 2026-06-08

### A-2:阿里云/银行 信贷管理系统(进件+贷后)
- **定位**:省级农信社/大零售信贷,管理从客户建档到资产保全的全流程
- **核心功能**:进件管理、评级授信、合同管理、押品管理、出账管理、贷后检查、风险分类、资产保全、档案管理
- **架构**:Java + Spring + 关系型数据库(传统银行 IT 栈)
- **可借鉴点**:①"押品管理"映射"材料版本管理" ②"风险分类"映射"议案审议结论" ③ 档案资料无纸化思路与我们 PDF 解析归档一致
- **不适用点**:五级风险分类(我们用通过/暂缓/否决简化);个贷/农贷产品工厂(我们单一项目类型)
- **评分**:业务匹配 3 / 技术栈 5 / 可借鉴 3 / 实现成本 3 = **14/20**
- **来源**:https://搜狐银行信贷管理 | 2026-06-08

### A-3:网商银行 / 蚂蚁金服 信贷系统
- **定位**:阿里云金融云 + 互联网银行,小微企业纯线上信贷
- **核心功能**:秒级授信、贷前反欺诈、贷中监控、贷后催收;全流程 AI 决策
- **架构**:金融云分布式核心 + 图计算 + 卫星遥感 + 大数据风控
- **可借鉴点**:①"申请→授信→放款"全在线 ② 贷后"逾期管家"对应"项目时点日程"
- **不适用点**:面向 C 端/小 B 端;风控规则与我们 P3 触发规则差异大;不能本地化(阿里云 only)
- **评分**:业务匹配 2 / 技术栈 2 / 可借鉴 2 / 实现成本 2 = **8/20**
- **来源**:https://e.mybank.cn | 2026-06-08

### A-4:启信宝 / 天眼查 企业征信
- **定位**:公开企业信息查询 + 关联族谱 + 风险监控(投前尽调)
- **核心功能**:企业工商/司法/投资/招投标 40+ 维度查询、关联族谱、风险监控、批量查询
- **架构**:数据采集 + 知识图谱 + 全文检索 + 移动端
- **可借鉴点**:①"关联族谱"映射"项目→议案→材料"三层关系图 ② 风险监控日报推送映射"时点到期提醒"
- **不适用点**:外部数据源对接(我们用内部档案);SaaS 化交付(我们要私有部署)
- **评分**:业务匹配 3 / 技术栈 3 / 可借鉴 3 / 实现成本 3 = **12/20**

## 类别 B:投委会/投后管理

### B-1:私募股权基金 投后管理体系(行业最佳实践)
- **定位**:VC/PE 基金对被投企业的"投后管理",募投管退四阶段之一
- **核心功能**:① 投资协议执行跟踪 ② 项目跟踪(月报/季报/年报) ③ 项目治理(参董事会) ④ 增值服务(战略/财务/HR 顾问) ⑤ 退出机制(IPO/并购/回购)
- **架构**:Excel + OA + 邮件,无统一系统(行业痛点)
- **可借鉴点**:① 季报/年报机制 → 我们的"投后跟踪报告"(M3 触发规则输出) ② 协议条款执行检查 → 我们的"时点日程自动提醒" ③ 增值服务清单 → 我们的"材料 checklist"
- **不适用点**:不涉及"投委会审议上会"流程(我们是投委会一方,他们是投后 GP 一方)
- **评分**:业务匹配 4 / 技术栈 2 / 可借鉴 5 / 实现成本 4 = **15/20** ⭐
- **来源**:https://搜狐/p/私募基金投后 | 2026-06-08

### B-2:嘉实基金 / 私募通 投后管理系统
- **定位**:大型公募/私募基金的组合管理 + 投后跟踪
- **核心功能**:组合配置、风险敞口、净值估算、信息披露、监管报送
- **架构**:商业软件 + 数据仓库
- **可借鉴点**:净值估算 → 我们"项目价值跟踪";信息披露 → 我们"投委会决议公开"
- **不适用点**:面向二级市场(我们是 PE/VC 一级市场);重监管报送
- **评分**:业务匹配 2 / 技术栈 2 / 可借鉴 2 / 实现成本 2 = **8/20**
- **来源**:https://docin.com/p-4590555687 | 2026-06-08

### B-3:VC/PE 内部 OA + Notion 实践

- **核心功能**:Notion 模板 + Excel + 飞书表格;无统一产品
- **架构**:文档协作 SaaS

## 类别 D:触发规则引擎

### D-1:Drools
- **定位**:JBoss 旗下 Java 规则引擎,基于 Rete 算法
- **核心功能**:DRL 规则文件、决策表、规则流、Kie API、Workbench
- **架构**:Java + Rete 匹配算法 + 复杂事件处理(CEP)
- **可借鉴点**:① 规则热加载(改 DRL 不重启) ② 决策表 → 我们的"触发规则 CRUD" ③ salience 优先级
- **不适用点**:重量级,学习曲线陡;小项目(我们 50 条规则)杀鸡用牛刀
- **评分**:业务匹配 3 / 技术栈 5 / 可借鉴 3 / 实现成本 2 = **13/20**
- **来源**:https://drools.org | 2026-06-08

### D-2:Aviator
- **定位**:轻量级 Java 表达式求值引擎(国产)
- **核心功能**:算术/逻辑/正则/三元运算;编译为 Java 字节码;支持 .av 脚本
- **架构**:ASM 字节码生成,无外部依赖
- **可借鉴点**:① 表达式 DSL 简单上手 ② 沙箱模式可限制函数 ③ 高性能(JIT 优化)
- **不适用点**:不是完整规则引擎(无 Facts/规则流);需要自己封装 Rule 抽象
- **评分**:业务匹配 4 / 技术栈 5 / 可借鉴 4 / 实现成本 4 = **17/20** ⭐⭐(首选)
- **来源**:https://github.com/killme2008/aviator | 2026-06-08

### D-3:Easy Rules
- **定位**:POJO 风格的轻量级 Java 规则引擎
- **核心功能**:@Rule/@Condition/@Action 注解;支持 MVEL/SpEL/JEXL 表达式
- **架构**:Java + MVEL
- **可借鉴点**:① 注解驱动,开发快 ② 支持 YML 规则文件(非程序员可改)
- **不适用点**:社区规模小;无热加载;无决策表 UI
- **评分**:业务匹配 3 / 技术栈 5 / 可借鉴 3 / 实现成本 4 = **15/20** ⭐
- **来源**:https://github.com/j-easy/easy-rules | 2026-06-08

### D-4:Camunda
- **定位**:BPMN 2.0 + CMMN + DMN 一站式流程平台
- **核心功能**:BPMN 流程定义、Cockpit 监控、Tasklist 用户任务、DMN 决策表
- **架构**:Java + Spring + 自研引擎
- **可借鉴点**:① DMN 决策表(可拖拽配置) ② 流程可视化
- **不适用点**:重型平台(50MB+),不适用"50 条规则"轻量场景
- **评分**:业务匹配 2 / 技术栈 4 / 可借鉴 2 / 实现成本 2 = **10/20**
- **来源**:https://camunda.com | 2026-06-08

## 类别 E:中文时点/日期抽取

### E-3:LLM-based 时点抽取(推荐)
- **定位**:用 LLM (GLM-4 / GPT-4) Prompt 抽取时间实体
- **核心功能**:Prompt:"从以下文本中抽取所有时间点,格式:YYYY-MM-DD | 事件描述"
- **架构**:我们已有 GlmService,加个 method 即可
- **可借鉴点**:① 零训练成本 ② 灵活 Prompt 控制 ③ 我们已经有智谱集成
- **不适用点**:速度慢(单次 1-2s);每次都要付费;长文档 chunk 处理
- **评分**:业务匹配 5 / 技术栈 5 / 可借鉴 5 / 实现成本 5 = **20/20** ⭐⭐⭐(本项目首选)
- **来源**:项目内 GlmService 复用 | 2026-06-08

### E-4:正则 + 规则(轻量兜底)
- **定位**:用正则表达式匹配常见中文时间格式
- **核心功能**:匹配"YYYY年MM月DD日"、"3天后"、"明年"、"Q1"、"每周一"等
- **架构**:Java Pattern
- **可借鉴点**:① 零成本 ② 离线可跑 ③ 对规范文档效果好
- **不适用点**:自然语言歧义("周三"是哪个周三?)
- **评分**:业务匹配 3 / 技术栈 5 / 可借鉴 3 / 实现成本 5 = **16/20** ⭐⭐(兜底首选)
- **来源**:常见实践 | 2026-06-08

## 类别 F:LLM Provider 抽象(OpenAI 兼容)

### F-1:Spring AI
- **定位**:Spring 官方 LLM 集成框架(2024 GA)
- **核心功能**:ChatClient 抽象、Prompt 模板、向量化、Function Calling、流式响应
- **架构**:Spring Boot Starter,自动配置,支持 OpenAI/Azure/Ollama/HuggingFace
- **可借鉴点**:① 与 Spring Boot 3.x 无缝集成(我们是 3.3) ② 标准化 ChatClient 接口
- **不适用点**:依赖较重(80MB+);版本号迭代快(0.7.0→0.8.1 跨度大)
- **评分**:业务匹配 4 / 技术栈 5 / 可借鉴 4 / 实现成本 3 = **16/20** ⭐⭐
- **来源**:https://spring.io/projects/spring-ai | 2026-06-08

### F-2:LangChain4j
- **定位**:Java 版 LangChain
- **核心功能**:链式调用、记忆管理、Tools 工具调用、RAG、向量库集成
- **架构**:模块化,15+ 嵌入存储、10+ LLM provider
- **可借鉴点**:① Tools 把 Java 方法注册为 LLM 工具 ② ChatMemory 自动管理上下文
- **不适用点**:过度设计(我们用 RAG 不需要 LangChain 全套);学习曲线陡
- **评分**:业务匹配 3 / 技术栈 5 / 可借鉴 3 / 实现成本 2 = **13/20**
- **来源**:https://github.com/langchain4j/langchain4j | 2026-06-08

### F-3:自研抽象(推荐,本项目轻量场景)

- **核心功能**:`LlmProvider` 接口,实现 `GlmProvider` / `OpenAiProvider` / `DashScopeProvider`
- **架构**:Spring @Service 多实现 + 配置切换
- **可借鉴点**:① 简单可控 ② 不引入新依赖 ③ 复用现有 GlmService
- **不适用点**:功能受限(无 Function Calling、无向量库集成)
- **评分**:业务匹配 5 / 技术栈 5 / 可借鉴 3 / 实现成本 5 = **18/20** ⭐⭐(本项目首选)
- **来源**:项目内 GlmService 演进 | 2026-06-08
