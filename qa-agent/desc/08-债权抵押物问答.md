# 债权抵押物问答经验

> **定位**：不良资产投委会档案里，**查债权相关信息是最常见任务**（债权标的、抵押物、估值/清偿状态）。  
> 本文沉淀 **提示词 + 会话记忆 + 引擎正则合成** 的完整链路，供后续扩展其他债权 Case 时复用。

---

## 1. 问句分型（先分类再路由）

| 类型 | 典型问句 | 证据来源 | 工具链 |
|---|---|---|---|
| **债权标的** | 「lmz 远期回购的债权标的是什么？」 | 回购协议 / 投资申请 | find → search（目标债权/岭兜）→ FINAL |
| **抵押物简述** | 「这个债权的抵押物是什么？」 | 「债权抵押物基本情况」章节 | search（**须含债权锚点**）→ FINAL |
| **抵押物细问** | 「xx债权下抵押物还剩哪些，初始估值分别是多少？」 | 同上 + 他项权证/贷款本金 | search（锚点+估值关键词）→ 清单式 FINAL |
| **多轮指代** | 上轮已答债权标的，本轮「这个债权…」 | session `last_debt_target` | 不重复 find；注入岭兜/建材二厂锚点 |

**硬规则**：抵押物属于**证据类**，走 `search_fulltext`，**不要**用 `get_project_business_data`（汇总表无正文）。

实现：`react_helpers.question_needs_*` + `memory.resolve_session_references`

## 2. 三层上下文（项目锁 + 债权主题）

```
用户问句
    │
    ├─ chat_session_context.project_code     ← 多轮「它/这个项目/抵押物」
    ├─ chat_session_context.last_debt_target ← 多轮「这个债权」
    └─ 问句内嵌「xx债权下/的」              ← 单轮即可解析，无需上轮记忆
```

| 模块 | 函数 | 作用 |
|---|---|---|
| `memory.py` | `load_context` / `save_context` | 读写 `last_debt_target`（需 SQL 迁移） |
| `memory.py` | `resolve_session_references` | 组合项目锁 + 债权锚点注入 |
| `memory.py` | `load_debt_target_from_history` | 无 DB 字段时从近期 assistant 回复反推 |
| `react_helpers.py` | `extract_debt_anchor_from_question` | 从「岭兜建材二厂债权下的…」解析债权名 |
| `engine.py` | `_prepare_session_context` | 启动轮次合并上述三路来源 |

## 3. 债权锚点解析（易错点）

### 3.1 问句内嵌债权名

正则：`([\u4e00-\u9fff]{2,40}债权)(?:下|的|中|项下)`

示例：

- 「岭兜建材二厂债权**下的**抵押物还剩哪些」→ `南安市岭兜建材二厂债权`
- 简称补全：问句含「岭兜建材二厂」+「债权」或「抵押」→ 规范化为完整债权名

### 3.2 指代词「这个债权」——不要当锚点写入 session

「这个债权的抵押物是什么？」会匹配 `这个债权下`，但 **必须过滤**：

- 演示性标签：`这个债权`、`该债权`、`上述债权` 等 → `extract_debt_anchor_from_question` 返回 `None`
- 真实锚点来自 **上轮** `last_debt_target` 或材料标题反推

### 3.3 检索 query 必须带债权锚点

| 错误 query | 后果 |
|---|---|
| 只搜 `抵押物` | 命中无关材料，答案变原文堆砌 |
| 只搜 `岭兜` | 可能仍太宽 |

**正确**（`evidence_search_query` 生成）：

```
岭兜 建材二厂 抵押物 土地 厂房 设备
```

细问再加：`估值 抵押金额 他项权证 贷款本金`

引擎提示（`append_step_hints`）会明确写：「债权抵押物问题 query 须含债权锚点…」

## 4. 抵押物细问识别

`question_needs_collateral_detail(question)` 为真时，走**清单式**答案。

触发词（须同时含「抵押物」类词）：

```
还剩哪些 | 剩余哪些 | 还剩 | 剩余 | 估值 | 评估价 | 抵押金额 | 初始价值 | 价值多少 | 分别
```

**注意**：

- 单独的「哪些」**不**触发细问（避免误伤）
- `_MATERIAL_EVIDENCE_RE` **不要**加裸 `剩余`，否则「剩余金额多少」会误入证据检索链

## 5. 材料正文抽取（引擎正则，不经 LLM）

段落锚点：`债权抵押物基本情况`（`react_helpers._COLLATERAL_SECTION_RE`）

### 5.1 按笔贷款拆条

正则：`第N笔贷款本金(\d+)万元，以(抵押描述)`

lmz 标准材料结构：

| 笔次 | 抵押物 | 本金 | 他项权证 | 清偿状态 |
|---|---|---|---|---|
| 第一笔 | 11.8亩工业土地 + 无证厂房 8617㎡ | 145万 | 抵押金额 354.18万 | 土地贷款本金已清偿，抵押物仍附属 |
| 第二笔 | 设备抵押 | 400万 | — | — |

函数链：

```
extract_collateral_items_from_texts
  → extract_collateral_items_from_hits
  → format_collateral_inventory_answer   # 细问
  → extract_collateral_from_texts        # 简述（；连接）
```

### 5.2 答案格式

**简述**（示例 2e）：

```
…抵押物包括：11.8亩工业土地抵押；上盖无证厂房8617平米；设备抵押。
```

**清单**（示例 2f）：

```
…剩余抵押物及初始估值：
- [1] …：354.18万元（他项权证记载抵押金额；贷款本金145万元）；对应贷款本金145万元已清偿…
- [2] 设备抵押：400万元（贷款本金）
```

合成入口：`synthesize_evidence_answer` / `try_finalize_evidence_from_search`（search 命中后立即合成，不等 LLM 下一步）。

## 6. 提示词（prompts.py）

### 6.1 场景路由表增补行

```
问「xx债权下抵押物还剩哪些/初始估值」
  → search_fulltext query 含债权锚点+抵押物+估值/抵押金额/他项权证
  → 按笔贷款提炼清单作答
```

已知 projectCode 时，利率/抵押/债权画像：**先** `get_project_analysis`，无快照再 `search_fulltext`。

### 6.2 Few-shot 必保留

| 编号 | 场景 | 要点 |
|---|---|---|
| **2d** | 远期回购债权标的 | search 含「目标债权」「岭兜」 |
| **2e** | 多轮「这个债权的抵押物」 | session 已锁 + query 含岭兜/建材二厂 |
| **2f** | 单轮「还剩哪些 + 初始估值」 | query 加长估值关键词；清单式 FINAL |

新增债权 Case 时：**优先补 Few-shot 原句**，再补引擎正则。
