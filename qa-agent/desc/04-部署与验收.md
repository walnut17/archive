## 4. 头号坑：写盘 ≠ 进程加载

**现象**：

- `push_update` 返回 200，`VERSION` 已是新 sha
- `/health` 仍返回**旧字段结构**（如只有 `version` 无 `git_sha`）
- `/v1/deploy/version` 404
- 利率路由等新逻辑**不生效**

**根因**：热更新覆盖磁盘，但 uvicorn 进程未真正重启；或自动重启脚本未 kill 旧 PID。

**识别**：

- 新 API：`pending_restart: true`（磁盘 sha ≠ 进程 sha）
- 旧 API：health 有 `version=新sha` 但无 `features`（进程仍是旧代码）

**处理**：125 **手工 stop → start -Force**（用户多次验证有效）

**自动重启失败根因（2026-06-15 实测）**：

| # | 问题 |
|---|---|
| 1 | `BackgroundTasks` 调度 `schedule_restart` 后进程未真正重启（`/deploy/restart` 40s 内 `process_started_at` 不变） |
| 2 | `stop-qa-agent.ps1` 只认 pid 文件，找不到则空跑，8001 旧进程继续服务 |
| 3 | `Popen(DETACHED)` 无日志，子进程失败不可见 |
| 4 | 热更新 zip **不含** `deploy/scripts/`，stop 脚本改进需 git pull |

**已修复**：同步 `Popen`、spawn/restart 日志、`/v1/deploy/restart-log`、按端口 kill 8001（内联在 `apply_update_and_restart.ps1`）

**代码侧改进**：

- `version_info._LOADED_GIT_SHA` 启动时冻结，避免误报
- `self_update` 热更后清 `__pycache__`
- `apply_update_and_restart.ps1` 必须 `-Force` + 验证 `process_started_at` 变化

## 5. TUI 验收

```powershell
cd D:\projects_new\projects-online\qa-agent
python tools\tui_repl.py --url http://182.168.1.125:8001
```

| 命令 | 用途 |
|---|---|
| 直接输入问句 | 流式验收工具链 |
| `/version` | 本地 git vs 远端运行版本 |
| `/health` | 看 features / git_sha |
| `/quiet` | 只看最终答案 |
| `/last` | 看完整步骤与来源 |

TUI 改 `tools/tui_repl.py` 后**重开 TUI**即可，**不依赖** 125 热更新（tools 在 zip 里但 TUI 在开发机跑）。
