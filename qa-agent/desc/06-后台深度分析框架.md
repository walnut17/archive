# 06 — 后台深度分析框架

## 目标

将**材料深度分析**从入库链路解耦：入库只做 Tika 解析 + 结构化表单预填；关键信息（项目层/资产层）由 qa-agent **后台异步**提取并持久化，供问答直读，避免每次讨论都全文检索。

## 架构

```
材料入库 (Java)                后台分析 (qa-agent)
─────────────                 ───────────────────
upload → Tika parse           AnalysisWorker 轮询
parsed_text 写入 DB     →     discover 指纹变化的项目
仅轻量预填 (可选)              enqueue analysis_job
                              按 analysis_template 调 LLM
                              写入 analysis_snapshot
                              同步 project_asset / project_fact
```

## 数据表

| 表 | 作用 |
|---|---|
| `analysis_template` | 可扩展分析模板（prompt + output_schema） |
| `analysis_job` | 任务队列 pending/running/success/failed |
| `analysis_snapshot` | 结构化结果快照（项目层/资产层） |
| `project_asset` | 底层资产实体（如债权） |
| `project_analysis_state` | 项目分析进度与材料指纹 |

迁移脚本：`deploy/sql/migrate_260615_analysis_framework.sql`

## 内置模板（可增不改码）

| code | scope | 说明 |
|---|---|---|
| `project.credit_inventory` | project | 识别全部债权标的 |
| `project.investment_structure` | project | 投资结构、交易形式、核心资产 |
| `project.interest_rate_schedule` | project | 利率/固定收益历程 |
| `asset.credit_profile` | asset | 单笔债权画像（债务人/抵押/法律状态等） |

新增模板：向 `analysis_template` 表 INSERT 一行即可；Worker 自动加载。

## 配置 (config.json)

```json
"qaAgent": {
  "analysisWorker": {
    "enabled": true,
    "pollIntervalSeconds": 30,
    "discoverBatch": 3
  }
}
```

## API

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/v1/analysis/status` | Worker 状态 + 队列统计 + 模板列表 |
| POST | `/v1/analysis/run-once` | 手动触发一轮 discover + 处理一个 job |
| POST | `/v1/analysis/enqueue` | 手动入队 `{project_code}` 或 `{project_id}` |
| GET | `/v1/analysis/projects/{code}` | 读取项目分析状态与 snapshots |

## 运行方式

1. **随 HTTP 服务自动启动**（`app/main.py` lifespan）
2. **独立进程**：`python scripts/analysis_worker.py`

## 扩展指南

1. 在 `analysis_template` 增加模板，设置 `scope=project|asset|material`
2. 如需自定义摘要，在 `extractor.summarize_result` 增加分支
3. 如需写入 `project_fact`，在 `mapper.sync_facts_from_snapshots` 增加映射
4. 后续可为 Agent 增加 `get_project_analysis` 工具，直读 snapshot 替代全文检索

## 与 Java 入库的关系

- Java `triggerAfterParse` 中的 `ExtractionEngine.extract()` **建议逐步降级**为仅预填或关闭
- 深度分析统一由 qa-agent Worker 负责，结果可被 Java 前端/API 直接读 `analysis_snapshot`
