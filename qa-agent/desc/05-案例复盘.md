# 案例复盘

> 不良资产投委会档案场景下的真实验收 Case，记录**问句 → 期望路由 → 实际根因 → 修复**。

## Case A：lmz 项目材料份数（T-0612-07/08）

### 问句

```
lmz项目下有多少份材料？
```

### 期望

```
find_project("lmz项目")
  → get_project_business_data(projectCode)
  → FINAL_ANSWER: 项目 xxx 下共 N 份材料。
```

### 曾出现的问题

| 现象 | 根因 |
|---|---|
| 重复 `find_project` 死循环 | LLM 未意识到已命中 |
| 重复 `get_project_business_data` | 未从 observation 收束到 FINAL |
| 答案失败 / 无 materialCount | `get_project_business_data` SQL 引用了不存在的 `p.stage` 字段 |
| 流式最终答案不醒目 | TUI 未分区展示 |

### 修复要点

1. **DB 对齐**：`get_project_business_data` 字段与 Java/DATABASE.md 一致
2. **引擎**：`maybe_upgrade_step` — find 命中后强制 get；get 有 count 后强制 FINAL
3. **恢复**：`try_recover_material_count_loop` — 重复调用时合成答案
4. **Prompt**：Few-shot 示例 2（lmz 几份材料）
5. **TUI**：`print_answer_box` 醒目最终答案

### 验收结果（125 重启后）

- PASS：「项目 lmz授信 (shtx26007) 下共 2 份材料。」
- 工具链正确，流式无 JSON 泄漏

## Case B：lmz 项目利率

### 问句

```
lmz项目的利率是多少？
```

### 期望

```
find_project
  → search_fulltext(query 含「利率」「固定收益」, projectCode 限定)
  → FINAL_ANSWER: 材料显示固定收益 X%
```

### 曾出现的问题

| 现象 | 根因 |
|---|---|
| 答不出 15% | 走了 `get_project_business_data`，**汇总表无利率正文** |
| 材料里有「固定收益15%」 | 数据在 `material_version.parsed_text`，必须全文检索 |
| 修了代码仍不生效 | 热更新写盘但进程未重启，旧 `engine.py` 仍在跑 |

### 曾出现的问题（续）

| 现象 | 根因 |
|---|---|
| search 已命中仍「多次尝试未找到」 | `observation` 被 2000 字截断，JSON 碎裂，`parsedExcerpt` 在字段末尾被裁掉，引擎无法解析摘录合成利率 |
| 连续两步相同 search_fulltext | LLM 重复选工具；recovery 也因 observation 解析失败而失效 |

### 修复要点（2026-06-15 二轮）

1. **`compact_search_hits_for_obs`**：observation 只存 `snippet`+标题，正文优先
2. **`try_finalize_evidence_from_search`**：`search_fulltext` 返回后用**完整 hits** 立即合成 FINAL，不再等 LLM 下一步
3. **`search_fulltext` LIKE 兜底**：修正 cursor 在 `with` 块外执行的 bug
4. **health 特性标记**：`features.post_search_finalize: 2026-06-15` 表示新逻辑已加载

### 验收要点

- 步骤中必须出现 `search_fulltext`
- observation 含材料摘录
- 答案含利率数值与引用来源
- `/health` → `features.evidence_routing: 2026-06-15`

## Case E：多轮债权标的 → 抵押物

### 问句（连续两轮）

```
1) lmz项目远期回购的债权标的是什么？
2) 这个债权的抵押物是什么？
```

### 期望

```
轮1: find → search(债权标的/岭兜) → 答「南安市岭兜建材二厂债权」
     → save_context(last_debt_target=…)

轮2: 不重复 find；resolve_debt_reference 注入「岭兜建材二厂」
     → search(岭兜 建材二厂 抵押物 …) → 简述抵押物清单
```

### 曾出现的问题

| 现象 | 根因 |
|---|---|
| 第二轮只搜「抵押物」 | 未带债权锚点，命中无关摘录 |
| 答案变材料原文粘贴 | 引擎未走 `extract_collateral_from_texts` |
| 第二轮不知道指代哪个债权 | `last_debt_target` 未落库或未从 history 恢复 |

### 修复要点

1. **DB**：`chat_session_context.last_debt_target` + 迁移 SQL
2. **memory**：`resolve_debt_reference` 对抵押物追问注入锚点
3. **引擎**：`FEATURE_COLLATERAL_ROUTING` + 抵押物合成
4. **Prompt**：Few-shot 2d / 2e

## Case F：抵押物还剩哪些 + 初始估值

### 问句

```
岭兜建材二厂债权下的抵押物还剩哪些，初始估值分别是多少？
```

### 期望

```
search_fulltext(query 含岭兜+抵押物+估值+抵押金额+他项权证+贷款本金)
  → 清单式 FINAL_ANSWER:
     [1] 土地+厂房：354.18万（他项权证）/ 本金145万；土地贷款已清偿…
     [2] 设备抵押：400万（贷款本金）
```

### 曾出现的问题

| 现象 | 根因 |
|---|---|
| 「这个债权」被写入 session 当锚点 | `extract_debt_anchor_from_question` 误匹配指代词 |
| 「剩余金额多少」走全文检索 | `_MATERIAL_EVIDENCE_RE` 含裸词「剩余」 |
| 只答抵押物名、无估值 | 未识别细问；未抽贷款本金/他项权证 |

### 修复要点

1. **问句分型**：`question_needs_collateral_detail`
2. **抽取**：`extract_collateral_items_from_texts`（笔贷款 + 他项权证 + 清偿句）
3. **过滤**：指代性「这个债权」不作为问句内锚点；`剩余` 不进 material_evidence 正则
4. **Prompt**：场景路由表 + Few-shot 2f
5. **文档**：[08-债权抵押物问答.md](./08-债权抵押物问答.md)

### 验收结果（125）

- PASS：两轮连贯 + 单轮细问清单含 354.18万 / 400万

## Case C：热更新后「版本对了但行为没变」

### 现象

- `push_update` 200，`VERSION=474c577`
- `check_version` 显示本地与远端 sha **一致**
- 但 `/v1/deploy/version` 404，`/health` 仍是旧 JSON 形状
- 利率 Case 仍走旧路由

### 根因

- 磁盘 `VERSION` 已更新，**uvicorn 进程未加载新 Python 模块**
- 旧版 health 每次 `read_version()` 读盘，造成「版本一致」的假象

### 修复与预防

1. `version_info`：进程启动冻结 `git_sha`，`pending_restart` 标识待重启
2. TUI `/version`、`push_update` 推送后检查 `pending_restart`
3. 运维：**手工 stop/start** 作为热更新后的标准步骤（直到自动重启 100% 可靠）

## 经验归纳（可复用到其他 LLM 档案场景）

1. **汇总数据 ≠ 正文证据**：表字段能答「多少份」，不能答「利率多少」
2. **Prompt + 引擎双轨**：Few-shot 教 LLM；`maybe_upgrade_step` 防典型错误
3. **问句正则要宽**：息率、费率、几份、多少个 等同义覆盖
4. **部署要验行为不验 sha**：sha 一致不代表进程加载了新代码
5. **每个 Case 固化测试**：`test_react_helpers.py` 用真实问句字符串回归
6. **债权抵押物**：汇总表答不了正文；多轮靠 `last_debt_target`；细问靠引擎清单抽取（见 [08](./08-债权抵押物问答.md)）
