## Step 0: 准备（部署前一次性配置）

### 0.1 SSH key（如果之前没装）

```bash
mkdir -p ~/.ssh
# 用项目方自己的 deploy key 放到 ~/.ssh/archive_deploy
chmod 600 ~/.ssh/archive_deploy
```

### 0.2 克隆仓库

```bash
cd /workspace
git clone -b minimax git@gitee.com:frisker/projects-online.git projects-online-deploy
cd projects-online-deploy

# 验证基线
git rev-parse HEAD
# 期望: 90b8616
```

## Step 1: 拉最新代码 + 编译

```bash
cd projects-online-deploy
git pull origin minimax   # 等价于 pull origin main（两分支同 commit）

# 后端编译
cd backend
mvn clean package -DskipTests -B
# 产物: backend/target/archive-*.jar
# 期望: BUILD SUCCESS, 3-15 分钟（首次装依赖慢）

# 前端构建
cd ../frontend
npm ci
npm run build
# 产物: frontend/dist/
# 期望: 0 错，10-20 秒
```

---

## Step 2: 部署到 182.168.1.125

### 2.1 上传 jar + dist

```bash
# 后端 jar
scp backend/target/archive-*.jar deploy@182.168.1.125:D:/projects-online/backend/

# 前端 dist
scp -r frontend/dist/* deploy@182.168.1.125:D:/projects-online/frontend/dist/
```

### 2.2 重启后端（WinSW）

```bash
ssh deploy@182.168.1.125
cd D:\projects-online
.\backend\stop.ps1   # 停老进程（如果 startup.ps1 里有 stop 子命令）
.\backend\startup.ps1  # 拉最新代码 + 重启 Spring Boot
```

> **如果 startup.ps1 没做自动 git pull**——手动跑一次：
> ```powershell
> cd D:\projects-online
> git pull origin minimax
> .\backend\stop.ps1
> .\backend\startup.ps1
> ```

### 2.3 重启 Caddy

```bash
ssh deploy@182.168.1.125
sc.exe stop caddy
sc.exe start caddy
# 或
net stop caddy && net start caddy
```

### 2.4 验证后端启动

```bash
curl http://182.168.1.125:8080/actuator/health
# 期望: {"status":"UP"}

curl http://182.168.1.125/
# 期望: Vue 前端 HTML（200 OK）
```

## 故障排查

| 报错 | 原因 | 解决 |
|---|---|---|
| `mvn package` 失败，缺 JSON/TEXT 列 | MySQL v2-schema 没跑 | 跑 `mvn flyway:migrate` 或手动建表 |
| 后端启动报 `mail health indicator` 503 | 没配真 SMTP | 已修（`2ae2b62`）禁了 mail indicator，重启即可 |
| `NetworkDictService` 调外网超时 | 业务方没给白名单 | 申请 `baike.baidu.com` + `wikipedia.org` 入站白名单 |
| PDF 导出字体中文方块 | 没装中文字体 | 业务方服务器装 Noto Sans CJK |
| 7 大场景某项失败 | 看后端 log | `tail -f D:/projects-online/logs/spring.log` |
